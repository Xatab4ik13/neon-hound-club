# Школа внутри клуба (PWA)

Переносим Школу HELLHOUND внутрь клуба на роуты `/club/school` и `/club/school/$instructorId`. Фокус — мобильная PWA-версия в стиле нативного приложения (как остальные разделы клуба: `/club/shop`, `/club/quests`). Десктоп внутри клуба — адаптация той же вёрстки (не копия лендинга). Лендинговые страницы `/school` и `/school/$instructorId` остаются как есть.

## Что делаем

### 1. Данные — общий источник
Оставляем `src/data/instructors.ts` единым источником. Ни лендинг, ни клуб не дублируют данные. Никаких правок контента (bio, цены, фото) в этой задаче.

### 2. Роут `/club/school` (список)
Заменяем текущую заглушку «Скоро» в `src/routes/club.school.tsx`.

Мобильная (основная) вёрстка:
- `PageHeader` с большим iOS-title «Школа» + подзаголовок «Учись у тех, кто катает».
- Сегмент-контрол `IOSSheet`-стиля: `Инструкторы` / `Онлайн-курсы` (как тумблер на лендинге, но в клубной стилистике — компактный pill, без plump-теней).
- Вкладка «Инструкторы» — вертикальный список карточек во всю ширину, а не сетка. Каждая карточка:
  - Фото 16:10 сверху со скруглением, поверх — плашки «Город» и «X лет стажа».
  - Имя крупно, ряд специализаций мелкими чипами, короткая строка «от N ₽ / занятие».
  - Тап по карточке → `/club/school/$instructorId`.
- Вкладка «Онлайн-курсы» — центрированный пустой стейт «Скоро», компактный (без огромного 9xl текста лендинга).

Десктоп внутри клуба:
- Та же логика, но карточки в сетке 2 колонки (сайдбар клуба уже занимает место).

### 3. Роут `/club/school/$instructorId` (профиль)
Новая страница в клубной стилистике. Секции сверху вниз:
- Hero: фото на всю ширину экрана (aspect 4/5), плавный градиент вниз, поверх — имя, город, стаж, чипы-специализации. Sticky кнопка `Записаться` снизу над таббаром на мобилке.
- Bio — обычный текстовый блок клуба (без plump-рамок).
- «Чему научишься» — вертикальный список `IOSList` строк (иконка + заголовок + текст), а не наклонённые plump-плашки.
- Курсы и цены — карточки как в `/club/shop` (плоские, скруглённые, `bg-card`), кнопка «Выбрать» на каждой.
- «Где занимаемся» — карта Яндекс (переиспользуем `src/lib/yandex-maps.ts`) + адрес и заметка. Одна карточка `bg-card`.
- Расписание — горизонтальный скролл дней (snap), слоты — компактные пилюли. Тап по свободному слоту открывает `IOSSheet` с заглушкой «скоро запись».
- Галерея — сетка 3 колонки (мобилка 2), тап открывает существующий `ImageViewer`.
- Финальный CTA — блок с кнопкой «Записаться».

Никаких `border-[3px]`, `shadow-[Npx_Npx_0_0]`, `rotate-*` из лендинга — клуб живёт в другой визуальной системе.

### 4. Навигация внутри клуба
- `MobileMoreSheet` уже ведёт на `/club/school` (пункт есть в `MORE_PATHS`) — оставляем.
- Десктопный сайдбар клуба — проверить, что ссылка «Школа» ведёт на `/club/school` (если нет — добавить).

### 5. SEO / мета
`noindex` для клубных страниц школы (клуб закрыт от индексации). Title/description для внутренней навигации браузера.

## Что НЕ делаем
- Не трогаем `/school` и `/school/$instructorId` на лендинге.
- Не меняем данные инструкторов, фото, цены.
- Не подключаем реальный API записи — слоты и запись остаются моками с заглушкой.
- Не трогаем бэкенд.

## Технические детали
- Файлы: правка `src/routes/club.school.tsx` (список), новый `src/routes/club.school.$instructorId.tsx` (профиль).
- Переиспользуем: `PageHeader`, `IOSSheet`, `IOSList`, `ImageViewer`, `yandex-maps.ts`.
- Тон-цвета инструкторов (`TONE_BG`) используем только как акцент чипов/CTA, не как заливку карточек.
- Sticky-кнопка «Записаться» на мобилке учитывает `MobileTabBar` (`bottom: calc(64px + env(safe-area-inset-bottom))`).
