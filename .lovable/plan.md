## Цель

Сделать в `/club/me` секцию «Гараж» с:
1. Стильными карточками байков (тёмная плашка, фото на прозрачном фоне с тонким glow-контуром).
2. Модалкой добавления/редактирования: гибридный выбор марки/модели/года с автокомплитом + ручной ввод для кастомных, загрузка фото байка.

Всё на фронте — без БД, без авторизации. Состояние в локальном `useState` + `localStorage` для персистентности между перезагрузками.

## Что делаем

### 1. База байков (автокомплит)

Источник — **NHTSA vPIC API** (бесплатно, без ключа):
- `GET /vehicles/GetMakesForVehicleType/motorcycle?format=json` → список всех мото-марок (~600 шт). Кешируем в `localStorage` на 30 дней.
- `GET /vehicles/GetModelsForMakeYear/make/{MAKE}/modelYear/{YEAR}?format=json` → модели для марки и года.
- Годы — генерим список 1980 — текущий год.

Поведение поля:
- Марка: combobox (shadcn `Command` + `Popover`) с поиском. Если ничего не подошло — снизу опция «➕ Добавить свою марку: <текст>».
- Модель: combobox, появляется после выбора марки и года. Зависит от выбранной марки/года, грузится по запросу. Та же кнопка «добавить свою модель» если нет в списке.
- Год: select от текущего года до 1980.

Если юзер выбрал «свою» марку/модель — флаг `custom: true` сохраняется в данных байка, отображается ярлык `CUSTOM` на карточке.

### 2. Загрузка фото байка

- В модалке — drag-n-drop зона или клик «Загрузить фото».
- Принимаем JPG/PNG/WEBP, до 5 МБ.
- Превью в модалке. Файл сохраняем как base64 dataURL в `localStorage` (на фронте, без бекенда).
- Если фото нет — карточка показывает дефолтный силуэт (один общий PNG-плейсхолдер в `src/assets/bikes/placeholder.png`, сгенерим через imagegen).
- В рамках этой итерации **фон НЕ убираем автоматически**. Юзер грузит как есть. Если хочет красивый прозрачный фон — пусть грузит PNG с прозрачностью. Автоматическое удаление фона — отдельная задача (нужен AI-сервис, обсудим потом).

### 3. Карточка байка (визуал)

- Тёмная плашка в стиле сайта (`bg-surface`, `border border-border`), при hover — `border-primary/40`.
- Фото байка сверху, занимает основную площадь, `object-contain` (чтобы прозрачный фон сохранялся), на фоне градиента `from-surface to-background`.
- Тонкий glow-контур по силуэту через CSS:
  ```text
  filter: drop-shadow(0 0 1px hsl(var(--primary)))
          drop-shadow(0 0 8px hsl(var(--primary) / 0.35));
  ```
  Работает на PNG с альфа-каналом. На фото с фоном — даёт лёгкое свечение по краям прямоугольника (не идеально, но не ломается).
- Текст: марка + модель крупно (font-display, uppercase), год моно мелко, прозвище курсивом если есть.
- Метаданные снизу: пробег, цвет, теги тюнинга (chips).
- Hover: лёгкое поднятие + усиление glow.
- Кнопки в углу: иконка-карандаш (редактировать), иконка-корзина (удалить).

### 4. Несколько байков

- Сетка: 1 колонка на мобиле, 2 колонки на десктопе.
- Если байков нет — пустая «полка»: пунктирная рамка, иконка + кнопка «Добавить байк».
- Кнопка «➕ Добавить байк» всегда видна над сеткой.

### 5. Поля карточки (как договаривались)

- Марка (обязательно)
- Модель (обязательно)
- Год (обязательно)
- Цвет
- Прозвище
- Пробег
- Дата покупки (datepicker)
- Тюнинг (список тегов, добавление через Enter)
- Фото

## Технические заметки

- NHTSA API не требует ключа, CORS разрешён, можно дёргать прямо с фронта.
- localStorage лимит ~5 МБ. Одно фото base64 в среднем 200-500 КБ. Хватит на 5-10 байков без проблем. Если упрёмся — потом перейдём на Lovable Cloud Storage.
- Никаких серверных функций, никакого Lovable Cloud в этой итерации.
- Используем shadcn `Command`, `Popover`, `Dialog`, `Input`, `Calendar`, `Badge` — всё уже есть в проекте.

## Что НЕ делаем сейчас

- Автоматическое удаление фона с фото — обсудим отдельно (нужен платный сервис либо AI).
- Хранение в БД и синхронизация между устройствами — потом, когда подключим авторизацию.
- Запрос характеристик мотоцикла (мощность, объём) — пока не надо, юзер видит только то что выбрал.
