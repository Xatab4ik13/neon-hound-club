## Цель

Сделать процесс «Добавить байк / Редактировать байк» в PWA без багов и в стиле дорогого iOS-приложения. Десктопную модалку не трогаем — там Dialog уже работает.

## Текущие проблемы (что я нашёл)

1. **Год выбирается через `<select>`** — в iOS PWA внутри vaul Drawer нативный picker иногда не открывается / закрывает sheet.
2. **Дата покупки через `<input type="date">`** — на iOS показывает уродский нативный picker, конфликтует с drawer.
3. **`ComboboxWithCustom` (марка/модель)** — мобильный пикер через `createPortal(body)` поверх vaul Drawer работает, но при закрытии иногда теряет фокус, body-scroll-lock конфликтует с drawer-scroll-lock.
4. **`window.confirm` при «закрыть с грязной формой»** — в standalone PWA на iOS блокирующий диалог часто игнорируется, sheet просто закрывается.
5. **Загрузка фото** — `<input type="file">` через ref на iOS PWA открывает медиатеку, но после возврата vaul иногда схлопывает sheet (т.к. visualViewport дёргается).
6. **CTA «Готово» в шапке IOSSheet** один на всё — нет визуальной обратной связи во время аплоада фото (только текст «...»).
7. **Тюнинг: «+ Добавить»** маленькая кнопка, на тапе на iOS не всегда срабатывает (нет touch-area 44px).
8. **Скролл формы** — при фокусе на инпут iOS поднимает контент, поле уезжает под клавиатуру, потому что у нас инпуты не скроллятся в видимую область.
9. **Заголовок IOSSheet** — нет кнопки «Отмена» слева, только свайп-ручка → юзер не понимает как выйти.

## Что меняем

### 1. `IOSSheet` — добавить опциональную кнопку слева

- Новый проп `cancelLabel?: string` (по умолчанию «Отмена»), `onCancel?: () => void`.
- Если `onCancel` задан — слева в шапке появляется кнопка-текст в iOS-стиле.
- Это убирает необходимость в `window.confirm` — закрытие через явный тап.

### 2. `BikeFormModal` — целиком переписать мобильную часть

Структура iOS-карточки настроек (как в iOS Settings / Apple Wallet):

```text
┌─────────────────────────────────┐
│  Отмена          Добавить       │  ← шапка sheet
├─────────────────────────────────┤
│                                 │
│   ┌─────────────────────┐       │
│   │   [фото / плюс]     │       │  ← большая фото-карточка
│   └─────────────────────┘       │
│                                 │
│  ОСНОВНОЕ                       │
│  ┌─────────────────────────┐    │
│  │ Марка        Yamaha  >  │    │  ← row → открывает picker
│  ├─────────────────────────┤    │
│  │ Модель       R6      >  │    │
│  ├─────────────────────────┤    │
│  │ Год          2024    >  │    │  ← wheel-picker, не <select>
│  └─────────────────────────┘    │
│                                 │
│  ДЕТАЛИ                         │
│  ┌─────────────────────────┐    │
│  │ Цвет         Чёрный     │    │  ← inline text input
│  │ Прозвище     Гончая     │    │
│  │ Пробег       18 400 км  │    │
│  │ Куплен       12.03.24 > │    │  ← кастомный date picker
│  └─────────────────────────┘    │
│                                 │
│  ТЮНИНГ                         │
│  ┌─────────────────────────┐    │
│  │ + Добавить              │    │
│  │   Akrapovič          ×  │    │
│  │   Pazzo levers       ×  │    │
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

- iOS-list секции: серая подложка `bg-white/[0.03]`, разделители `border-white/[0.05]`, скруглённые края секций.
- Row-минимум **44px** (Apple HIG).
- Заголовки секций — мелкий uppercase tracked, серый.

### 3. Новый компонент `IOSWheelPicker` (для года)

- Полноэкранный sheet поверх формы, скроллящийся «барабан» как iOS native.
- Решает баг с `<select>` внутри Drawer.

### 4. Новый компонент `IOSDateField`

- Open → IOSSheet с тремя «барабанами» (день/месяц/год).
- На десктопе остаётся `<input type="date">`.

### 5. `ComboboxWithCustom` — поправить мобильный пикер

- Убрать `document.body.style.overflow = "hidden"` (конфликт с vaul).
- Поднять z-index до 300, чтобы точно был над vaul (он 81).
- Использовать `inputMode="search"` + `enterkeyhint="done"`.
- Кнопка «Готово» в шапке picker'а, не «Назад».

### 6. Фото-поле в PWA

- Большая карточка 16:9 с круглой кнопкой `+` по центру (iOS-стиль).
- Тап → нативный picker (input file как сейчас).
- Если фото есть → круглая кнопка `⋯` в углу → action sheet «Заменить / Удалить».
- Состояние «загружается» — overlay со спиннером поверх превью.

### 7. Тюнинг-секция

- Inline-row «+ Добавить тюнинг» → IOSSheet с одним инпутом + «Добавить» в шапке.
- Каждая модификация — row с ×-кнопкой 44px справа.

### 8. Грязная форма + закрытие

- Кнопка «Отмена» в шапке → если форма грязная, показываем **IOSConfirm** (у нас уже есть `src/components/ios/IOSConfirm.tsx`), а не `window.confirm`.
- Свайп-вниз за ручку тоже триггерит этот же confirm.

### 9. Сохранение

- Кнопка «Добавить»/«Сохранить» в шапке.
- Во время сабмита: текст меняется на «Сохраняем…», вся форма disabled (полупрозрачный overlay), нельзя свайпнуть закрыть.
