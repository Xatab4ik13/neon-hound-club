## Принятые решения (из ответов)

- **Главная для участника:** та же страница + плашка «вы в клубе» в hero и avatar-меню в Header.
- **Покупка мерча:** только для авторизованных. Гость на «Купить» → редирект на `/login` с возвратом.
- **Билеты в чекауте:** не используются, только рубли (билеты остаются для розыгрышей).
- **Тон «О клубе»:** клуб по подписке + бонусы участникам (розыгрыши = стимулирующая акция среди подписчиков Hell Pass).

## Этапы

### Этап 1. Фундамент: состояние «гость / участник» + Header
- Завести лёгкий контекст `useViewer()` (mock, без бэка): `isAuthed`, `nick`, `tier`, `tickets`. Источник — `ME` из `src/data/profile.ts`. Для теста добавить toggle (флаг в localStorage), пока нет реальной авторизации.
- `Header`:
  - гость → как сейчас (Войти);
  - участник → avatar-dropdown: ник, тир, билеты, ссылки «Мой гараж» / «Hell Pass» / «Выйти».
- На главной (`/`) для участника — короткая плашка над hero «Вы в клубе → продолжить в кабинете», без подмены контента.

### Этап 2. Публичные информационные страницы
Каждая страница = своя route с собственным `head()` (title/description/og).

1. **`/club/hell-pass`** (публичная) — берём контент из `club.hell-pass.index.tsx`, выкидываем sidebar и любые «личные» виджеты (баланс, кнопки «активировать»). Оставляем: 3 тира, сравнение, FAQ, CTA «Войти и оформить». Кабинетная версия остаётся под `/_authenticated` (или текущим путём).
2. **`/school`** (публичная) — аналогично из `club.school.tsx`. Курсы Город / Трек / Падение, цены, программа, CTA «Войти и записаться». 
3. **`/about`** — новая. Кто такие, что за проект, ссылки на YouTube / Telegram / Instagram / Twitch (placeholder URL — спрошу у тебя позже). Юридический блок: «Hell Pass — платная клубная подписка. Розыгрыши — стимулирующая акция среди подписчиков, отдельная плата за участие не взимается, выигрыш не зависит от внесения дополнительных платежей». Без слова «лотерея».
4. **`/news`** — новая. Лента карточек (мерч-анонсы, итоги розыгрышей, новости клуба). Тип данных в `src/data/news.ts`. Список + страница статьи `/news/$slug` с полноценным `head()` для SEO (h1, description, og:image из обложки). 2-3 мок-статьи для старта.

### Этап 3. Корзина и чекаут
- **Архитектура:**
  - Cart-store в localStorage + лёгкий React context (`useCart()`). Работает и у гостя (можно собирать корзину), но «Оформить» доступно только авторизованным.
  - `/cart` — публичная, видна всем. Список позиций, qty, удалить, итого.
  - `/checkout` — под `_authenticated` (или ручной guard с редиректом на `/login?redirect=/checkout`).
- **Чекаут:**
  - Шаги в одну страницу: 1) Получатель (имя, телефон, email — префилл из профиля); 2) Доставка (СДЭК / Почта России / самовывоз — пока без интеграции, выбор + адрес); 3) Оплата (карта/СБП — заглушка-кнопка «Оплатить», без реального шлюза, можно потом подключить Stripe/ЮKassa).
  - После «оплаты» → `/checkout/success` с номером заказа, начисленные кешбэк-билеты (1 за 200₽).
- **Кешбэк-билеты:** показываем превью в правой колонке чекаута («Получите N билетов после оплаты»).

### Этап 4. Полировка
- Footer: ссылки на новые страницы (`/about`, `/news`, `/club/hell-pass`, `/school`).
- SEO: `head()` на каждой публичной странице, sitemap-friendly структура.
- Меню Header: проверить активные состояния для новых маршрутов.

## Технические заметки

- Все новые route-файлы — плоское именование TanStack (`about.tsx`, `news.tsx`, `news.$slug.tsx`, `cart.tsx`, `checkout.tsx`, `checkout.success.tsx`).
- Дизайн: переиспользуем существующие токены, секции и компоненты (`Hero`, карточки розыгрышей, sticker-стиль). Никаких новых цветов.
- Реальный бэк не подключаем — всё на mock-данных и localStorage. Когда созреет — Lovable Cloud + Supabase auth + RLS.
