# Инструкторы — новый тип аккаунта

Новая роль `instructor` (5 персональных аккаунтов: Станислав, Семён, Никита, Паша, Хайкс). Работаем только в PWA. Всё на моках — бэк подключим позже.

## 1. Что видит инструктор в PWA

Такой же интерфейс, как у обычного пользователя, с одним отличием в нижнем табе:

```text
[ Лента ] [ Магазин ] [ Билеты ] [ Школа ] [ Ещё ]
                                    ^^^^^
                                    вместо Гаража
```

- **Школа** заменяет **Гараж** в таббаре.
- **Гараж** переезжает в «Ещё» (шторка `MobileMoreSheet`).
- В остальном меню всё как у обычного юзера.

### Экран «Школа» у инструктора

Список чатов с учениками — тот же паттерн, что у Вани в `/blogger/chats`:
- аватар ученика, ник, превью последнего сообщения, время, бейдж непрочитанных;
- поиск по нику сверху;
- тап по строке → открывается чат 1-на-1, точно такой же UI, как VIP-чат (`club.vip-chat.tsx`): сообщения слева/справа, стикеры, зелёная подсветка инпута, springy-анимация новых сообщений.

Инструктор в этом чате играет роль «блогера» (аналог Hell в VIP-чате).

## 2. Что видит обычный пользователь

На страницах инструкторов (`/club/school/$instructorId`) появляется кнопка **«Записаться»**.

Тап по кнопке → снизу поднимается bottom-sheet во весь экран с чатом с этим конкретным инструктором. UI чата — 1:1 копия VIP-чата, только собеседник = выбранный инструктор (аватар, имя, город в шапке шита).

- Шит открывается свайпом-снизу с pull-to-close сверху.
- История чата с этим инструктором сохраняется между открытиями (мок в `localStorage`).
- Стикеры, отправка, springy-анимация — как в VIP-чате.

## 3. Данные (моки)

- `src/data/instructor-accounts.ts` — маппинг `instructorSlug → { userId, nick, avatar }` для 5 инструкторов. Роль `instructor` в мок-профиле текущего юзера включается через dev-переключатель (как раньше делали для blogger), чтобы ты мог зайти под инструктором и посмотреть интерфейс.
- `src/data/instructor-chats-mock.ts` — мок-треды инструктора с 3–5 учениками + история сообщений на каждого. Хранится в `localStorage`, чтобы отправленные тобой сообщения не пропадали при перезагрузке.
- Со стороны ученика треды с инструкторами лежат в том же сторе, просто перевёрнутая ролью запись.

Никаких новых бэкенд-миграций/роутов в этой итерации — договорились, всё на моках.

## 4. Файлы (что создаём / трогаем)

Новое:
- `src/data/instructor-accounts.ts`
- `src/data/instructor-chats-mock.ts` (стор + хуки чтения/записи)
- `src/components/instructor/InstructorMobileTabBar.tsx` — копия обычного таббара, но `School` вместо `Garage`
- `src/components/instructor/InstructorMoreSheet.tsx` — «Ещё» с добавленным пунктом «Гараж»
- `src/routes/instructor.school.index.tsx` — список чатов (по образцу `blogger.chats.index.tsx`)
- `src/routes/instructor.school.$userId.tsx` — окно чата (по образцу `blogger.chats.$userId.tsx` / `club.vip-chat.tsx`)
- `src/components/school/BookInstructorChatSheet.tsx` — bottom-sheet с чатом для ученика

Правим:
- `src/routes/club.tsx` (или где выбирается таббар) — если у текущего юзера роль `instructor`, показываем `InstructorMobileTabBar` вместо стандартного.
- `src/routes/club.school.$instructorId.tsx` — добавляем кнопку «Записаться», открывающую `BookInstructorChatSheet`.
- `src/hooks/use-viewer.tsx` — поддержка роли `instructor` (мок-флаг), чтобы можно было переключиться и посмотреть интерфейс.

## 5. Как ты это проверишь

1. В dev-переключателе роли выбираешь `instructor` (например, Станислав) → таббар меняется, вместо Гаража появляется Школа, внутри — список чатов с учениками (мок).
2. Открываешь чат → пишешь ученику, работает как VIP-чат.
3. Переключаешься обратно на обычного юзера → заходишь на `/club/school/stanislav` → жмёшь «Записаться» → снизу поднимается чат со Станиславом, пишешь сообщение — оно появится в его списке чатов, когда переключишься обратно на инструктора.
