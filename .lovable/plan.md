# Админ-панель HELLHOUND

CRM-интерфейс по адресу `/admin`. Отдельная вёрстка от публичного сайта: нейтральная палитра (slate/zinc), плотные таблицы, sidebar слева, переключатель тема светлая/тёмная. 1 роль `admin` (через таблицу `user_roles` + `has_role()`).

## Структура маршрутов

```
/admin/login              — вход (email+пароль)
/admin/_layout            — sidebar + topbar + ThemeToggle (защищён beforeLoad)
  /admin                  — Дашборд
  /admin/raffles          — Розыгрыши (список)
  /admin/raffles/new      — создать
  /admin/raffles/$id      — редактировать
  /admin/users            — Пользователи (список + поиск/фильтры)
  /admin/users/$id        — карточка: профиль, ранг, билеты, Hell Pass, история
  /admin/shop             — Товары (список)
  /admin/shop/new
  /admin/shop/$id
  /admin/shop/categories  — категории/подкатегории + витрина на главной
  /admin/tickets          — Билеты: тарифы покупки, массовая выдача, история операций
  /admin/news             — Новости/SEO-статьи
  /admin/economy          — Экономика (P&L + операции + доли)
  /admin/hell-ai          — заглушка «В разработке» (Этап 2)
  /admin/school           — заглушка «В разработке»
  /admin/cdek             — заглушка «В разработке» (Этап 2)
  /admin/settings         — настройки команды (приглашение admin)
```

## Sidebar (порядок)

Дашборд · Розыгрыши · Пользователи · Магазин · Билеты · Новости · Экономика · Hell AI · Школа · СДЭК · Настройки

## Этап 1 — что реально реализуем

### 1. Каркас + авторизация
- Включить Lovable Cloud (если ещё не включён).
- Таблицы: `user_roles` (enum `app_role` = `admin`), функция `has_role()`.
- `_admin` layout: `beforeLoad` редиректит, если нет роли `admin`.
- ThemeToggle через `data-theme` атрибут + отдельный набор токенов в `styles.css` (`--admin-bg`, `--admin-fg`, `--admin-border`, `--admin-accent`) — не ломает фронт сайта.
- Sidebar collapsible, на мобиле — Sheet.

### 2. Дашборд (полезный, не «ради дашборда»)
Карточки сверху:
- Выручка за месяц (магазин + Hell Pass) с трендом ±%.
- Активные подписки Hell Pass по тирам (Silver/Gold/Platinum).
- Новых пользователей за неделю.
- Билетов выдано / списано за неделю.
- Активных розыгрышей и сумма билетов в банке.

Списки ниже:
- Последние 10 заказов (статус, сумма, СДЭК-трек когда появится).
- Розыгрыши, которым осталось <48ч.
- Подписки, истекающие в ближайшие 7 дней.

### 3. Розыгрыши
Поля: название, описание (rich text / markdown), фото (multi-upload в storage `raffle-images`), цена в билетах, дата старта, дата окончания, лимит на участника, статус (draft/active/finished). Кнопки: опубликовать, завершить вручную, выгрузить список участников CSV.

### 4. Пользователи
Таблица: ник, email, дата регистрации, ранг, тир Pass, билетов на счету, последний вход. Поиск по нику/email/телефону. Карточка пользователя:
- Личная инфа (всё, что заполнял в профиле + мотоцикл).
- Кнопки: сменить ранг (select из `data/ranks.ts`), выдать N билетов (с причиной — в журнал), подарить Hell Pass на X дней (Silver/Gold/Platinum), забанить.
- История: покупки, участия в розыгрышах, начисления/списания билетов.

### 5. Магазин
**Товары:** название, slug, цена ₽, цена в билетах (опц.), описание, галерея, категория, подкатегория, остаток, статус (active/draft/archived), флаг «на витрине главной» (с порядком сортировки).
**Категории:** CRUD, drag-and-drop порядок, подкатегории дочерним списком. Отдельная вкладка «Витрина» — выбор и порядок до 6 товаров для главной.

### 6. Билеты
- Тарифы прямой покупки (X билетов за Y₽) — CRUD.
- Конвертация кешбэка (1 билет за 200₽) — настройка.
- Массовая выдача: выбор пользователей (по фильтру: тир Pass, ранг, ручной список) + сумма + причина → запись в журнал.
- Журнал операций с фильтрами и экспортом CSV.

### 7. Новости / SEO
Создание статьи: title, slug, description (meta), og:image, обложка, тело (markdown), теги, дата публикации, статус. SEO-поля рядом с превью сниппета Google.

### 8. Экономика (по ответам выше — всё четыре пункта)
Структура:
- **Операции:** таблица всех движений денег. Поля: дата, тип (доход/расход), категория, сумма, валюта, контрагент, комментарий, источник (manual / auto:shop / auto:hellpass). Импорт/экспорт CSV.
- **Авто-учёт:** на каждый успешный заказ магазина и продление Pass — автозапись в операции (категория «Магазин» / «Подписки»).
- **Категории расходов:** CRUD. Дефолтный набор: Продакшн, Себестоимость мерча, Призы, Налоги, Реклама, Эквайринг, Прочее.
- **Партнёры и доли:** CRUD партнёров (имя, %). Кнопка «Распределить за период» → берёт чистую прибыль (доходы − расходы) за выбранный месяц и считает выплату каждому. Сохраняется как «Расход / Выплата партнёру».
- **P&L отчёт:** график помесячно (доходы стэком по источникам, расходы по категориям, чистая прибыль линией). Экспорт PDF/CSV.

### 9. Заглушки
`Hell AI`, `Школа`, `СДЭК` — страница с заголовком и пометкой «В разработке. Появится на этапе 2».

## Технические детали
- Стек: TanStack Start (как сейчас), Lovable Cloud (Supabase), shadcn `Sidebar`, `Table`, `Dialog`, `Form` (react-hook-form + zod).
- Темы: отдельные токены `--admin-*` в `src/styles.css`, переключение через `data-theme="dark"` на `<html>` под маршрутом `/admin`.
- Все мутации — через `createServerFn` + `requireSupabaseAuth` + проверка `has_role(uid, 'admin')` внутри handler'а. RLS на всех таблицах.
- Загрузка картинок — bucket'ы `raffle-images`, `product-images`, `news-images` (public read, write только admin).
- Аудит — таблица `admin_audit` (actor, action, entity, entity_id, diff jsonb, created_at).
